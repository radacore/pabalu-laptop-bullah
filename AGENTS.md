# AGENTS.md — Pabalu Laptop

Operational guidance for AI agents working in this repo. Skip generic Laravel/React advice; this file only documents what is hard to infer from filenames.

## Stack

- Laravel 13.7 + PHP 8.3 (CI matrix also runs 8.4, 8.5)
- Inertia v3 + React 19 + TypeScript 5.7, Tailwind CSS v4 (Vite plugin, no `tailwind.config.js`)
- Laravel Fortify (auth) + custom `EnsureUserHasRole` middleware
- Pest v4 (PHP) + Playwright (e2e)
- MySQL for dev and prod (not sqlite)
- shadcn/ui (`new-york` style, lucide icons), Wayfinder for typed frontend route helpers

## Setup

```bash
composer install
cp .env.example .env && php artisan key:generate
npm install                       # use npm, not pnpm (see Gotchas)
php artisan storage:link          # required for logo + laptop photo uploads
php artisan migrate --seed
composer dev                      # boots server + queue + pail + vite together
```

Default seeder login: `admin@pabalu.com` / `password`.

MySQL database `db_pabalu_laptop` must exist (utf8mb4). `phpunit.xml` overrides to `sqlite::memory:` only inside tests — do not assume sqlite works for dev.

## Routing & roles

Three route tiers in `routes/web.php` (single file, `routes/settings.php` is separate):

1. **Public** — `/`, `/shop`, `/laptops/{laptop}`, `/services/track/{trackingCode}` (throttled `10,1`).
2. **`auth + verified + role:admin,staff`** — dashboard, services CRUD, service updates/parts, laptop photos, global search.
3. **`auth + verified + role:admin`** — laptops CRUD, customers, financial transactions, master data, website settings.

Roles: `admin` | `staff` | `customer`. Middleware alias `role` → `App\Http\Middleware\EnsureUserHasRole` (registered in `bootstrap/app.php`).

Cookies exempted from encryption: `appearance`, `sidebar_state`.

## Generated files (do not edit by hand)

These are gitignored and regenerated by tooling — edit the source config, not the output:

- `resources/js/actions/`, `resources/js/routes/`, `resources/js/wayfinder/` — Wayfinder route helpers (regenerated by Vite via `@laravel/vite-plugin-wayfinder`).

- `resources/js/actions/index.ts` and similar — Wayfinder entrypoints.

`eslint.config.js` already ignores all of the above.

## Public pages design system (Hallmark Cobalt)

The four public routes — `/`, `/shop`, `/laptops/{laptop}`, `/services/track/{trackingCode}` — share a single design system based on Hallmark's Cobalt theme (modern-minimal genre). Source of truth: Hallmark skill at `.agents/skills/hallmark/`. Admin panel uses a separate Material 3 system (do not apply Cobalt tokens there).

**Hard rules (do not break):**

- **6px radius buttons.** Every interactive button uses `rounded-btn` (6px). Two variants: filled (`bg-accent text-accent-ink`) and outlined (`border border-rule text-ink hover:border-accent`). NO pill buttons on CTAs.
- **Hairlines do the work.** 1px `border-rule` separates all surfaces. No shadow hierarchy — depth comes from borders.
- **Single accent color discipline.** `oklch(58% 0.20 256)` (electric cobalt) for links/active states and CTA fills. Used sparingly (< 5% of viewport).
- **Body text discipline.** 16px / line-height 1.5 / weight 400 / color `oklch(34% 0.018 257)` (ink-2). Never pure `#000` for body copy.
- **Display headlines** use `cobalt-display` utility (Space Grotesk, -0.025em tracking, weight 600). Left-aligned, never centred hero.
- **Mono labels** use `cobalt-caption` (JetBrains Mono, uppercase, 0.06em tracking) for specs, status, metadata.
- **Section rhythm.** Alternating bands: `bg-paper` (cool near-white canvas) and `bg-surface` (slightly tinted). Section padding `py-16 md:py-20`.
- **Max content width** is `max-w-[980px]`. Hero can extend wider.
- **Left-aligned hero.** Title + lede left, product image right. NOT centred.
- **Three font families.** Space Grotesk (display), Inter (body), JetBrains Mono (code/labels). Shipped via Bunny Fonts.

**Tokens (defined in `resources/css/app.css` `@theme`):**

- Colors: `--color-paper` oklch(98.5% 0.004 250), `--color-ink` oklch(24% 0.02 258), `--color-ink-2` oklch(34% 0.018 257), `--color-accent` oklch(58% 0.20 256), `--color-rule` oklch(88% 0.008 250), `--color-graphite` oklch(22% 0.016 260), `--color-surface` oklch(96% 0.006 250)
- Radius: `--radius-card` 10px, `--radius-btn` 6px, `--radius-pill` 980px (search inputs only)
- Shadow: `--shadow-card` (minimal, 1px lift), `--shadow-product` (product imagery only)

**Typography utilities** (`@utility` block in `app.css`): `cobalt-display`, `cobalt-heading-lg`, `cobalt-heading`, `cobalt-subheading`, `cobalt-body-lg`, `cobalt-body`, `cobalt-body-sm`, `cobalt-caption`. Use these instead of raw font sizes.

**Don't reach for in public pages:**

- shadcn/ui primitives (`resources/js/components/ui/*`) — admin-only
- `Material Symbols` font — admin-only
- `bg-slate-*`, `bg-white`, `bg-blue-*` — use Cobalt tokens only
- `rounded-xl` / `rounded-2xl` — use `rounded-card` (10px) or `rounded-btn` (6px)
- `shadow-sm` / `shadow-md` on UI — hairlines only
- Apple tokens (`bg-cloud`, `bg-bone`, `text-graphite`, `text-fog`, `text-apple-blue`, `bg-button-blue`, `apple-*` utilities) — removed, use Cobalt equivalents

**Shared shell:** `resources/js/components/public-layout.tsx` exports `PublicPage`, `PublicHeader`, `PublicFooter`. Header is 48px sticky bordered nav with `bg-paper/80 backdrop-blur-md`. Footer is `bg-surface` with `border-t border-rule`. Wrap any new public page in `<PublicPage website={...} title="...">`.

## Code conventions worth knowing

- Frontend uses `import/order` with alphabetized groups (builtin → external → internal → parent → sibling → index) and `@typescript-eslint/consistent-type-imports` enforced as `separate-type-imports`. New files must follow this.
- Blank lines around control statements (`if`, `return`, `for`, `try`, etc.) enforced by `@stylistic/padding-line-between-statements`. Lint will flag missing padding.
- ESLint uses flat config. `@stylistic/brace-style` is `1tbs`, `curly` is `all` — `if (x) do();` is rejected.
- `react/react-in-jsx-scope`, `react/prop-types`, `react/no-unescaped-entities` are off.
- React Compiler is enabled via `babel-plugin-react-compiler` in `vite.config.ts`. Write code as if the compiler runs (avoid manual `useMemo`/`useCallback` that fight it; rely on memoization at boundaries).
- Vite ships Bunny Fonts (Inter, Outfit, Space Grotesk, JetBrains Mono) — not Google Fonts. Do not add font links.
- Tailwind classes sort automatically via `prettier-plugin-tailwindcss`. Class helpers: `clsx`, `cn`, `cva` are recognized by Prettier.
- Prettier: 4-space indent, single quotes, 80 col, semicolons. YAML uses 2-space (`.editorconfig` override). `resources/js/components/ui/*` and `resources/views/mail/*` are prettierignored.
- EditorConfig: 4 spaces (2 for yml), LF, UTF-8, final newline.

## Tests

- **PHP (Pest)** in `tests/Feature/` and `tests/Unit/`. Entry: `tests/Pest.php`, `tests/TestCase.php`. Test env vars live in `phpunit.xml`.
- **Playwright e2e** in `tests/*.spec.js` (currently `admin-sidebar.spec.js`). Browser spec: Playwright's bundled Chromium/Firefox/WebKit via `npx playwright test`.
- Factories in `database/factories/`. Seeders run in order: `MasterData → User → Business → Testimonial → WebsiteSetting` (`database/seeders/DatabaseSeeder.php`).

## CI

Two workflows under `.github/workflows/`:

- `lint.yml` — runs `composer lint`, `npm run format`, `npm run lint`.
- `tests.yml` — matrix `[8.3, 8.4, 8.5]`, runs `./vendor/bin/pest`.

Both trigger on push/PR to `develop`, `main`, `master`, `workos`. Dependabot config in `.github/dependabot.yml`.

## Gotchas

- `pnpm-workspace.yaml` exists alongside `package-lock.json`. **Use npm.** The workspace file only exists to public-hoist `@inertiajs/core` for stable SSR resolution; the lockfile and CI all use npm.
- Don't run `php artisan test` as the only check — Pint won't run. Use `composer test`.
- `tests/` was previously incorrectly gitignored; the current `.gitignore` correctly excludes only root-level debug files. Do not add `tests/` to gitignore.
- `php artisan storage:link` must run on every fresh checkout or logo/photo uploads 404.
- `routes/web.php` is the source of truth; `routes/settings.php` is for Fortify settings routes. Don't merge them.
- `bootstrap/app.php` is the Laravel 11+ style entry — middleware aliases and route registration live here, not in `Kernel.php`.
- `prd.md` in root is the project spec; `README.md` has user-facing flow diagrams and ERD.
- Generated Wayfinder helpers (in `resources/js/{actions,routes,wayfinder}`) are gitignored — don't `git add -f` them.
